================================================================================
USING THE SYSTEM
================================================================================

The frontend form has:
  - GitHub Repository URL
  - Target Branch
  - Folder Name
  - Local Workspace Path  ← You control this!

Local Workspace Path options:

  Relative path (resolved relative to project root):
    ./GitHub          → PROJECT_ROOT/GitHub/repo
    ./repos           → PROJECT_ROOT/repos/repo
    repos             → PROJECT_ROOT/repos/repo

  Absolute path:
    /Users/prativa/Documents/Code        → /Users/.../Code/repo
    C:\Users\Username\Code\Repos         → C:\Users\...\Repos\repo

  Home directory:
    ~/code/repos                         → /Users/prativa/code/repos/repo
    ~\code\repos                         → C:\Users\prativa\code\repos\repo

  Leave blank (uses default):
    (empty)           → PROJECT_ROOT/repos/repo

================================================================================
VERIFICATION
================================================================================

After cloning, verify:

1. Check logs show absolute path:
   [CloneAgent] localStorageResolved (absolute)=/path/to/clone/location
   [CloneAgent] os.path.isabs(localStorageResolved)=True

2. Check folder exists:

   macOS / Linux:
   ls -la ./GitHub/shopstack-platform_testing

   Windows (Command Prompt):
   dir .\GitHub\shopstack-platform_testing

   Windows (PowerShell):
   Get-ChildItem -Path .\GitHub\shopstack-platform_testing

3. Verify it's in the right location (not PROJECT_ROOT/backend/C:/)

================================================================================
TROUBLESHOOTING
================================================================================

Backend won't start:
  macOS: Make sure .venv/bin/activate was run
  Windows: Make sure .venv\Scripts\activate was run
  Check Python version: python --version (need 3.13+)

Frontend won't start:
  Check Node is installed: node --version (need 18+)
  Check npm packages installed: npm list
  Try: npm install && npm run dev

Repos cloning to wrong location:
  Check logs for: [CloneAgent] localStorageResolved
  Should be an absolute path, not containing "C:\" or backslashes on macOS
  Verify "Local Workspace Path" in frontend form is filled correctly
